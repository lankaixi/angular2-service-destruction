Instructions

  1. Open the Chrome task manager and identify the process for this tab
  2. In the Chrome task manager, make sure the Javascript Memory column is displayed
  3. Note down the JS Memory, it should be around 23Mb
  4. In each test, we'll push numbers 1 to 1M into an array and observe the JS Memory usage

Test 1 (Page A)

Service Provider Retains No Array Reference & Memory is Released by GC on Destory()

  1. In the application, navigate to Page A and click on the "Load Up Array" button to create a large array
  2. You'll see the JS Memory allocation climb to about 1.3Gb, then drop back to about 838Mb
  3. In the "Destruction Panel", click the first button to destroy the bootstrapped application
  4. Run the garbage collector or wait 1-2 minutes - memory will be released

Test 2 (Page B)

Service Provider Retains Array Reference, Memory is NOT Released on Destroy()

  1. In the application, navigate to Page B. The Page B Service will automatically load up the array.
  2. You'll see the JS Memory allocation climb to about 1.3Gb, then drop back to about 838Mb
  3. In the "Destruction Panel", click the first button to destroy the bootstrapped application
  4. Note in the task manager that the JS Memory allocated has not changed.
